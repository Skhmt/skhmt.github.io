TAPIC.js Reference


TAPIC.js

Latest version

Github repo

TAPIC.js is a purely javascript framework for accessing Twitch chat and most of the Twitch API. It was created for use in KoalaBot mostly, but it has many other useful applications.

Why should you use TAPIC.js when TMI.js exists? TAPIC.js is lighter weight (~41kb for tmi.js vs ~10kb for tapic.js) and provides roughly the same functionality. Use either or both or neither.

Its intended use is for client-side web apps, CLI/BrowserSource pages for OBS, and NW.js/Electron desktop apps. It will also work with Node.js: npm install tapic.

The example test.html shows how to use TAPIC.js in a simple chatroom reading website. The example requires inputting your own clientid and oauth to work.

The example test.js shows how to use TAPIC.js in a Node.js app. This example isn't as feature-complete as the .html version.

TAPIC.setup( clientid, oauth, callback )

Sets up TAPIC.js. You must run this before anything works.

Parameters

  • clientid (string) - The public client id of your app.
  • oauth (string) - The oauth token of the user you want to be logged in. Does not include oauth:.
  • callback (function) - Callback function has one parameter, the username.

Example

  • TAPIC.setup( clientid, oauth, function( username ) {
  console.log( 'The username is: ' + username );
} );

TAPIC.runChat( channel, callback )

Sets the initial channel to connect to, connects to it and the group (whisper) server, and runs various other set-up things.

Parameters

  • channel (string) - The channel name to connect to.
  • callback (function) - for when the channel is running.

Example

  • TAPIC.runChat( channel, function() { } );

Listener - raw

Every RAW TMI message from the standard chat server. You most likely won't be using this unless you need to parse for something that TAPIC.js doesn't already have a listener for.

Callback functions

  • res (string) - The raw message.

Example

  • TAPIC.listen('raw', function( res ) { console.log( res ); } );

Listener - message

A regular message (PRIVMSG in IRC). This includes actions (/me).

Callback functions

  • res.from (string) - The username of the person who sent the message.
  • res.text (string) - The text of the message.
  • res.color (string) - The color of the user name.
  • res.emotes (string) - The emote id and character locations. See: https://github.com/justintv/Twitch-API/blob/master/IRC.md#privmsg.
  • res.action (boolean) - True if it is an action (/me), false if it is a regular message.
  • res.streamer (boolean) - True if the streamer (channel name) sent the message, false if it is anyone else.
  • res.mod (boolean) - True if the user that sent the message is a moderator, false if not.
  • res.sub (boolean) - True if the user that sent the message is a subscriber, false if not.
  • res.turbo (boolean) - True if the user that sent the message has turbo, false if not.
  • res.badges (array) - Array of badges, such as 'broadcaster/1', 'subscriber/1', and 'warcraft/alliance'.
  • res.room_id (integer) - The chatroom ID of the room the message was sent to.
  • res.user_id (integer) - The Twitch ID number of the user that sent the message.

Example

  • TAPIC.listen('message', function( res ) {
  var output = (res.mod ? '<img src="http://chat-badges.s3.amazonaws.com/mod.png">' : '') +
    (res.sub  ?'<img src="' + TAPIC.getSubBadgeUrl() + '">' : '') +
    (res.turbo ? '<img src="http://chat-badges.s3.amazonaws.com/turbo.png">' : '') +
    (res.streamer ? '<img src="http://chat-badges.s3.amazonaws.com/broadcaster.png">' : '') +
    '<strong style="color: ' + res.color + ';">' +
    res.from +
    '</strong> ' +
    (res.action ? '<span style="color: ' + res.color + ';">' : ':  ') +
    res.text +
    (res.action ? '</span>' : '' );
  document.querySelector('#chat').innerHTML += output;
} );

Listener - whisper

A whisper sent to the user.

Callback functions

  • res.from (string) - The user that sent the whisper.
  • res.to (string) - The recipient of the whisper (the bot name).
  • res.text (string) - The text of the whisper.
  • res.color (string) - The color of the sender's name.
  • res.turbo (boolean) - True if the sender is turbo, false if not.
  • res.message_id (integer) - Message id.
  • res.thread_id (integer) - Thread id.
  • res.user_id (integer) - User id.
  • res.badges (array) - Array of badges, such as 'broadcaster/1', 'subscriber/1', and 'warcraft/alliance'.

Example

  • TAPIC.listen('whisper', function( res ) {
  var output = (res.turbo ? '<img src="http://chat-badges.s3.amazonaws.com/turbo.png">' : '') +
    '<strong style="color: ' + res.color + ';">' +
    res.from +
    '</strong>' +
    ' > ' +
    '<strong>' + res.to + '</strong> : ' +
    res.text;
  document.querySelector('#chat').innerHTML += output;
} );

Listener - echoChat

Echos chat messages sent by the bot.

Callback functions

  • res (string) - The text of the chat message.

Listener - echoWhisper

Echos whispers sent by the bot.

Callback functions

  • res.to (string) - The target of the whisper.
  • res.text (string) - The text of the whisper.

Listener - notice

Notices from the standard chat server. For example, the response of the /mods command.

Callback functions

  • res (string) - The notice.

Listener - join

When a user joins your channel. Fires for every user in the channel when you enter, so be careful on how you use this. This doesn't seem to be real-time, unlike real IRC.

Callback functions

  • res (string) - The entering user's name.

Listener - part

When a user leaves your channel. This doesn't seem to be real-time, unlike real IRC.

Callback functions

  • res (string) - The parting user's name.

Listener - clearUser

This is sent when a user is timed out or banned or "purged" by a moderator. The default action should be to remove or hide all of that user's previous chat text.

Callback functions

  • res.name (string) - The cleared user's name.
  • res.reason (string) - The optional reason for timeout.
  • res.duration (string) - The length of the timeout in seconds.

Listener - clearChat

This is sent when a moderator wants to purge all of the chat. The default action should be to remove or hide all of the previous chatroom text.

Listener - host

This is sent when the user (not the channel) is hosted. If the logged in user is not the broadcaster, this will not be an accurate notification of hosts.

To get host notifications if you're not logged in as the broadcaster, you need to use:
http://tmi.twitch.tv/hosts?include_logins=1&target=[user id number] (Won't work on a client-side browser - no CORS/JSONP support)
or
https://decapi.me/twitch/hosts?channel=[channel name] (CORS support. Thanks Decicus)

Callback functions

  • res (string) - The user that hosted you.

Listener - follow

This is sent when a user follows the channel. The limitation is about 100 follows per minute. Any more might get lost.

Callback functions

  • res (string) - The user that followed the channel.

Listener - sub

Subscription notification.

Callback functions

  • res (string) - The user that subscribed.

Listener - subMonths

Resubscription and months and maybe message notification.

Callback functions

  • res.name (string) - The name of the person that resubscribed.
  • res.months (integer) - Number of months subscribed. Alternatively, number resubscribes + 1.
  • res.message (string) - Optional resub message.

Listener - subsAway

Number of subscribers since you've been offline.

Callback functions

  • res (integer) - Number of subscribers.

Listener - roomstate

The roomstate options set on a room.

Callback functions

  • res.lang (string) - The language of the room. Often left blank.
  • res.r9k (boolean) - True if the room is in r9k mode, false if not.
  • res.slow (boolean) - True if the room is in slow mode, false if not.
  • res.subs_only (boolean) - True if the room is in subs (and mods) only mode, false if not.

Example

  • TAPIC.listen('roomstate', function( res ) {
  var output = 'Roomstate options: ';
  if ( res.lang ) output += ' lang:' + res.lang;
  if ( res.r9k == 1 ) output += ' r9k';
  if ( res.slow == 1 ) output += ' slow';
  if ( res.subs_only == 1 ) output += ' subs-only';
  if ( !res.lang && res.r9k != 1 && res.slow != 1 && res.subs_only != 1 ) {
    output += ' none.';
  }
  document.querySelector('#chat').innerHTML += output;
} );

TAPIC.sendChat( text )

Sends a message to the channel. Actions (/me, /host, etc) work.

Parameters

  • text (string) - The message to send to the channel.

TAPIC.sendWhisper( user, text )

Sends a whisper to a specific user.

If you use whispers, it might be a good idea to get your app whitelisted.

Parameters

  • user (string) - The user to whisper a message to.
  • text (string) - The message to whisper.

TAPIC.changeChannel( channel )

Changes channel you're in. This is quicker than TAPIC.runChat(), but might not work if you change from a regular channel to an event server channel.

Parameters

  • channel (string) - The channel to change to.

TAPIC.runCommercial( length )

Requires the channel_commercial permission. Runs a commercial.

Parameters

  • length (integer) - The seconds to run the commercial. If empty, defaults to 30 seconds.

TAPIC.setStatusGame( status, game )

Sets the channel's status (title) and game played.

Parameters

  • status (string) - The status (title) for the stream.
  • game (string) - The game being played in the stream.

Example

  • TAPIC.setStatusGame('Making a Twitch Javascript Framework #programming', 'Creative');

TAPIC.closeChat()

Closes the current chat, including the group (whisper) server connection, opened by TAPIC.runChat(). This is automatically invoked by TAPIC.runChat() before it opens a new chat, so it won't be necessary very often.

TAPIC.emit( event, data )

Triggers listeners on event and gives them data. Custom events can also be defined and used.

Parameters

  • event (string) - The event name, such as 'message' or 'action'.
  • data (any) - The data for the event - see the Listeners section for propper formatting.

TAPIC.getUsername()

Gets the username that's logged in to TAPIC.js. This may not be the same as the channel you're in.

Returns string - The username.

TAPIC.getChannel()

Gets the channel that you're in.

Returns string - The channel name.

TAPIC.getGame()

Gets the game being played. For example: DayZ, Creative, etc.

Returns string - The game name.

TAPIC.getStatus()

Gets the status (title) of the stream.

Returns string - The stream status.

TAPIC.getFollowerCount()

Gets the number of followers of the stream.

Returns integer - The number of followers.

TAPIC.getTotalViewCount()

Gets the total (cumulative) number of viewers of the stream.

Returns integer - The total number of viewers of the stream.

TAPIC.getCurrentViewCount()

Gets the current number of viewers watching the stream.

Returns string - The current number of viewers in the stream.

TAPIC.getCreatedAt()

The time the stream started.

Returns string - The time the stream was created at.

TAPIC.getLogo()

Gets the channel's 300x300px logo URL.

Returns string - The logo URL.

TAPIC.getVideoBanner()

Gets the channel's offline video banner/image URL.

Returns string - The offline video banner URL.

TAPIC.getProfileBanner()

Gets the channel's user profile banner URL.

Returns string - The user profile banner URL.

TAPIC.getDisplayName()

Gets the user's display name, which includes capitalization, while getUsername always reutrns lowercase.

Returns string - The display name of the user.

TAPIC.getColor()

Gets the user's preferred coloring in the '#ABC123' hex format.

Returns string - The username color.

TAPIC.getEmoteSets()

Gets the user's emote sets in comma-separated format.

Returns string - The emote sets.

TAPIC.getMod()

Returns a 1 if the user is a moderator in the channel or a 0 if not.

Returns integer - 1 if moderator, 0 if not.

TAPIC.getSub()

Returns a 1 if the user is a subscriber to the channel or a 0 if not.

Returns integer - 1 if subscriber, 0 if not.

TAPIC.getTurbo()

Returns a 1 if the user is paying for Turbo or a 0 if not.

Returns integer - 1 if turbo, 0 if not.

TAPIC.getUserType()

The user's usertype. For example, 'staff'.

Returns string - The user type.

TAPIC.isFollowing( user, target, callback )

Checks if user is following target.

Parameters

  • user (string) - The user.
  • target (string) - The target streamer.
  • callback (function) - callback.isFollowing is a boolean for if the user is following the target or not. Callback.dateFollowed is a string for when the user started following the target, but is null if she is not following the target.

Example

  • var folUser = 'skhmt';
var folTarget = 'letsmakefriends';
TAPIC.isFollowing( folUser, folTarget, function( res ) {
  if ( res.isFollowing ) console.log( folUser + ' has been following ' + folTarget + ' since: ' + res.dateFollowed );
  else console.log( folUser + ' is not following ' + folTarget + '.' );
} );

TAPIC.isSubscribing( user, channel, callback )

Requires the channel_check_subscription permission. Checks if user is subscribed to channel.

Parameters

  • user (string) - The user to check.
  • channel (string) - The channel to check against.
  • callback (function) - Callback.isSubscribing is a boolean, true if subscribing, false if not. Callback.dateSubscribed is the date subscribed.

Example

  • var subUser = 'skhmt';
var subChannel = 'letsmakefriends';
TAPIC.isSubscribing( subUser, subChannel, function( res ) {
  if ( res.isSubscribing ) console.log( subUser + ' has been subbed to ' + subChannel + ' since: ' + res.dateSubscribed );
  else console.log( subUser + ' is not subbed to ' + subChannel + '.' );
} );

TAPIC.isPartner()

Checks if the channel is partnered with Twitch.tv.

Returns boolean - True if a partner, false if not.

TAPIC.getSubBadgeUrl()

Gets the URL for the subscriber badge used in front of chat messages.

Returns string - URL for the subscriber badge. Empty if not partnered.

TAPIC.isOnline()

Checks if the channel is online.

Returns boolean - True if the channel is online, false if it's offline.

TAPIC.getFps()

Gets the FPS the channel is currently running at.

Returns string - The frames per second of the channel.

TAPIC.getVideoHeight()

Gets the video height (480, 720, 1080, etc) of the channel.

Returns integer - The video height.

TAPIC.getDelay()

Gets the delay setting of the stream. This doesn't show actual delay from Twitch.tv to users, just the intentionally added delay.

Returns string - The delay.

TAPIC.getChatters()

Gets the current chatters in the chatroom.

Returns object - 

{
  "moderators": ["user01", "user02", ...],
  "staff": ["user11", "user12", ...],
  "admins": ["user21", "user22", ...],
  "global_mods": ["user31", "user32", ...],
  "viewers": ["user41", "user42", ...]
}
