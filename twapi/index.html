TWAPI.js Reference


TWAPI.js

Latest version

Github repo

TWAPI.js is a purely javascript framework for accessing Twitch chat and most of the Twitch API. It was created for use in KoalaBot mostly, but it has many other useful applications.

Why should you use TWAPI.js when TMI.js exists? No idea. Use either or both or neither.

Its intended use is for client-side web apps, CLI/BrowserSource pages for OBS, and NW.js/Electron desktop apps. It will not work with vanilla node.js due to the use of document.(event functions) and browser-based websockets.

The example test.html shows how to use TWAPI.js in a simple chatroom reading website. The example requires inputting your own clientid and oauth to work.

TWAPI.setup( clientid, oauth, callback )

Sets up TWAPI.js. You must run this before anything works.

Parameters

  • clientid (string) - The public client id of your app.
  • oauth (string) - The oauth token of the user you want to be logged in. Does not include oauth:.
  • callback (function) - Callback function has one parameter, the username.

Example

  • TWAPI.setup( clientid, oauth, function( username ) {
	console.log( 'The username is: ' + username );
} );

TWAPI.runChat( channel, callback )

Sets the initial channel to connect to, connects to it and the group (whisper) server, and runs various other set-up things.

Parameters

  • channel (string) - The channel name to connect to.
  • callback (function) - for when the channel is running.

Example

  • TWAPI.runChat( channel, function() { } );

Listener - twapiRaw

Every RAW TMI message from the standard chat server. You most likely won't be using this unless you need to parse for something that TWAPI.js doesn't already have a listener for.

Callback functions

  • .detail (string) - The raw message.

Example

  • document.addEventListener('twapiRaw', function( e ) { console.log(e.detail); } );

Listener - twapiRawGroup

Every RAW TMI message from the group (whisper) chat server. You most likely won't be using this unless you need to parse for something that TWAPI.js doesn't already have a listener for.

Callback functions

  • .detail (string) - The raw message.

Listener - twapiMsg

A regular message (PRIVMSG in IRC) from the standard chat server. This includes actions (/me).

Callback functions

  • .detail.from (string) - The username of the person who sent the message.
  • .detail.text (string) - The text of the message.
  • .detail.color (string) - The color of the user name.
  • .detail.emotes (string) - The emote id and character locations. See: https://github.com/justintv/Twitch-API/blob/master/IRC.md#privmsg.
  • .detail.action (boolean) - True if it is an action (/me), false if it is a regular message.
  • .detail.streamer (boolean) - True if the streamer (channel name) sent the message, false if it is anyone else.
  • .detail.mod (boolean) - True if the user that sent the message is a moderator, false if not.
  • .detail.sub (boolean) - True if the user that sent the message is a subscriber, false if not.
  • .detail.turbo (boolean) - True if the user that sent the message has turbo, false if not.

Example

  • document.addEventListener('twapiMsg', function( e ) {
	var output = (e.detail.mod ? '<img src="http://chat-badges.s3.amazonaws.com/mod.png">' : '') +
		(e.detail.sub  ?'<img src="' + TWAPI.getSubBadgeUrl() + '">' : '') +
		(e.detail.turbo ? '<img src="http://chat-badges.s3.amazonaws.com/turbo.png">' : '') +
		(e.detail.streamer ? '<img src="http://chat-badges.s3.amazonaws.com/broadcaster.png">' : '') +
		'<strong style="color: ' + e.detail.color + ';">' +
		e.detail.from +
		'</strong> ' +
		(e.detail.action ? '<span style="color: ' + e.detail.color + ';">' : ':  ') +
		e.detail.text +
		(e.detail.action ? '</span>' : '' );
	document.querySelector('#chat').innerHTML += output;
} );

Listener - twapiWhisper

A whisper sent via the group chat channel.

Callback functions

  • .detail.from (string) - The user that sent the whisper.
  • .detail.to (string) - The recipient of the whisper (the bot name).
  • .detail.text (string) - The text of the whisper.
  • .detail.color (string) - The color of the sender's name.
  • .detail.turbo (boolean) - True if the sender is turbo, false if not.
  • .detail.message_id (integer) - Message id.
  • .detail.thread_id (integer) - Thread id.
  • .detail.user_id (integer) - User id.

Example

  • document.addEventListener('twapiWhisper', function( e ) {
	var output = (e.detail.turbo ? '<img src="http://chat-badges.s3.amazonaws.com/turbo.png">' : '') +
		'<strong style="color: ' + e.detail.color + ';">' +
		e.detail.from +
		'</strong>' +
		' > ' +
		'<strong>' + e.detail.to + '</strong> : ' +
		e.detail.text;
	document.querySelector('#chat').innerHTML += output;
} );

Listener - twapiNotice

Notices from the standard chat server. For example, the response of the /mods command.

Callback functions

  • .detail (string) - The notice.

Listener - twapiNoticeGroup

Notices from the group (whisper) chat server.

Callback functions

  • .detail (string) - The notice.

Listener - twapiJoin

When a user joins your channel. Fires for every user in the channel when you enter, so be careful on how you use this. This doesn't seem to be real-time, unlike real IRC.

Callback functions

  • .detail (string) - The entering user's name.

Listener - twapiPart

When a user leaves your channel. This doesn't seem to be real-time, unlike real IRC.

Callback functions

  • .detail (string) - The parting user's name.

Listener - twapiClearUser

This is sent when a user is timed out or banned or "purged" by a moderator. The default action should be to remove or hide all of that user's previous chat text.

Callback functions

  • .detail (string) - The cleared user's name.

Listener - twapiClearChat

This is sent when a moderator wants to purge all of the chat. The default action should be to remove or hide all of the previous chatroom text.

Listener - twapiHost

This is sent when the user (not the channel) is hosted. If the logged in user is not the broadcaster, this will not be an accurate notification of hosts.

To get host notifications if you're not logged in as the broadcaster, you need to use:
http://tmi.twitch.tv/hosts?include_logins=1&target=[user id number] (Won't work on a client-side browser - no CORS/JSONP support)
or
https://decapi.me/twitch/hosts?channel=[channel name] (CORS support. Thanks Decicus)

Callback functions

  • .detail (string) - The user that hosted you.

Listener - twapiFollow

This is sent when a user follows the channel. The limitation is about 100 follows per minute. Any more might get lost.

Callback functions

  • .detail (string) - The user that followed the channel.

Listener - twapiSub

Subscription notification.

Callback functions

  • .detail (string) - The user that subscribed.

Listener - twapiSubMonths

Resubscription and months notification.

Callback functions

  • .detail.name (string) - The name of the person that resubscribed.
  • .detail.months (integer) - Number of months subscribed. Alternatively, number resubscribes + 1.

Listener - twapiSubsAway

Number of subscribers since you've been offline.

Callback functions

  • .detail (integer) - Number of subscribers.

Listener - twapiRoomstate

The roomstate options set on a room.

Callback functions

  • .detail.lang (string) - The language of the room. Often left blank.
  • .detail.r9k (boolean) - True if the room is in r9k mode, false if not.
  • .detail.slow (boolean) - True if the room is in slow mode, false if not.
  • .detail.subs_only (boolean) - True if the room is in subs (and mods) only mode, false if not.

Example

  • document.addEventListener('twapiRoomstate', function( e ) {
	var output = 'Roomstate options: ';
	if ( e.detail.lang ) output += ' lang:' + e.detail.lang;
	if ( e.detail.r9k ) output += ' r9k';
	if ( e.detail.slow ) output += ' slow';
	if ( e.detail.subs_only ) output += ' subs-only';
	if ( !e.detail.lang && !e.detail.r9k && !e.detail.slow && !e.detail.subs_only ) {
		output += ' none.';
	}
	document.querySelector('#chat').innerHTML += output;
} );

TWAPI.sendChat( text )

Sends a message to the channel. Actions (/me, /host, etc) work.

Parameters

  • text (string) - The message to send to the channel.

TWAPI.sendWhisper( user, text )

Sends a whisper to a specific user.

If you use whispers, it might be a good idea to get your app whitelisted.

Parameters

  • user (string) - The user to whisper a message to.
  • text (string) - The message to whisper.

TWAPI.changeChannel( channel )

Changes channel you're in. This is quicker than TWAPI.runChat(), but might not work if you change from a regular channel to an event server channel.

Parameters

  • channel (string) - The channel to change to.

TWAPI.getUsername()

Gets the username that's logged in to TWAPI.js. This may not be the same as the channel you're in.

Returns string - The username.

TWAPI.getChannel()

Gets the channel that you're in.

Returns string - The channel name.

TWAPI.getGame()

Gets the game being played. For example: DayZ, Creative, etc.

Returns string - The game name.

TWAPI.getStatus()

Gets the status (title) of the stream.

Returns string - The stream status.

TWAPI.getFollowerCount()

Gets the number of followers of the stream.

Returns integer - The number of followers.

TWAPI.getTotalViewCount()

Gets the total (cumulative) number of viewers of the stream.

Returns integer - The total number of viewers of the stream.

TWAPI.getCurrentViewCount()

Gets the current number of viewers watching the stream.

Returns string - The current number of viewers in the stream.

TWAPI.getCreatedAt()

The time the stream started.

Returns string - The time the stream was created at.

TWAPI.getLogo()

Gets the channel's 300x300px logo URL.

Returns string - The logo URL.

TWAPI.getVideoBanner()

Gets the channel's offline video banner/image URL.

Returns string - The offline video banner URL.

TWAPI.getProfileBanner()

Gets the channel's user profile banner URL.

Returns string - The user profile banner URL.

TWAPI.isFollowing( user, target, callback )

Checks if user is following target.

Parameters

  • user (string) - The user.
  • target (string) - The target streamer.
  • callback (function) - callback.isFollowing is a boolean for if the user is following the target or not. Callback.dateFollowed is a string for when the user started following the target, but is null if she is not following the target.

Example

  • var folUser = 'skhmt';
var folTarget = 'letsmakefriends';
TWAPI.isFollowing( folUser, folTarget, function( res ) {
	if ( res.isFollowing ) console.log( folUser + ' has been following ' + folTarget + ' since: ' + res.dateFollowed );
	else console.log( folUser + ' is not following ' + folTarget + '.' );
} );

TWAPI.isSubscribing( user, channel, callback )

Requires the channel_check_subscription permission. Checks if user is subscribed to channel.

Parameters

  • user (string) - The user to check.
  • channel (string) - The channel to check against.
  • callback (function) - Callback.isSubscribing is a boolean, true if subscribing, false if not. Callback.dateSubscribed is the date subscribed.

Example

  • var subUser = 'skhmt';
var subChannel = 'letsmakefriends';
TWAPI.isSubscribing( subUser, subChannel, function( res ) {
	if ( res.isSubscribing ) console.log( subUser + ' has been subbed to ' + subChannel + ' since: ' + res.dateSubscribed );
	else console.log( subUser + ' is not subbed to ' + subChannel + '.' );
} );

TWAPI.isPartner()

Checks if the channel is partnered with Twitch.tv.

Returns boolean - True if a partner, false if not.

TWAPI.getSubBadgeUrl()

Gets the URL for the subscriber badge used in front of chat messages.

Returns string - URL for the subscriber badge. Empty if not partnered.

TWAPI.runCommercial( length )

Requires the channel_commercial permission. Runs a commercial.

Parameters

  • length (integer) - The seconds to run the commercial. If empty, defaults to 30 seconds.

TWAPI.isOnline()

Checks if the channel is online.

Returns boolean - True if the channel is online, false if it's offline.

TWAPI.getFps()

Gets the FPS the channel is currently running at.

Returns string - The frames per second of the channel.

TWAPI.getVideoHeight()

Gets the video height (480, 720, 1080, etc) of the channel.

Returns integer - The video height.

TWAPI.getDelay()

Gets the delay setting of the stream. This doesn't show actual delay from Twitch.tv to users, just the intentionally added delay.

Returns string - The delay.

TWAPI.getChatters()

Gets the current chatters in the chatroom.

Returns object - 

{
	"moderators": ["user01", "user02", ...],
	"staff": ["user11", "user12", ...],
	"admins": ["user21", "user22", ...],
	"global_mods": ["user31", "user32", ...],
	"viewers": ["user41", "user42", ...]
}

TWAPI.setStatusGame( status, game )

Sets the channel's status (title) and game played.

Parameters

  • status (string) - The status (title) for the stream.
  • game (string) - The game being played in the stream.

Example

  • TWAPI.setStatusGame('Making a Twitch Javascript Framework #programming', 'Creative');

TWAPI.closeChat()

Closes the current chat, including the group (whisper) server connection, opened by TWAPI.runChat(). This is automatically invoked by TWAPI.runChat() before it opens a new chat, so it won't be necessary very often.
